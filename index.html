GitHub flow - Vad är det? Hur använder man det? Vilka viktiga delar är inkluderade i begreppet?

GitHub flow är något som vi använder för att få en smidig och enkelt användande av GitHub oftast när man jobbar i grupper men kan även tillämpas när man jobbar själv. Man anväder det genom att skapa nya branchar, gör sina nya ändringar i den branshen, commitar upp den med tydliga kommentarer så andra lätt kan tolka vad man förändrat, öppnar upp en pullrequest, där brukar man lägga till reviewers så andra i gruppen kan kolla igenom det man gjort, när andra kollat igenom så allt ser bra ut godkänner de sin review och du som skapat pull requesten brukar då göra en merge så den branchen slås ihop med main branchen.

Flexbox - Vad är det? Hur fungerar det?

Flexbox är något som används för att göra enkla responsiva sidor. När man jobbar med flexbox jobbar man med olika axlar för att styra innehållet med antingen row eller column. Man har en parent till innehållet och sedan har den parent sina children inuti sig. På detta sätt kan man styra de olika children inuti parent på ett enklare sätt som t.ex. positionering, storlekar på de olika boxarna osv.

DOM (Document Object Model) - Vad är det? Hur manipulerar man DOM?

Används för att kunna gå in och manipulara innehåll i dokumentent, man kan genom JavaScript gå in i specifika delar och ändra innehållet när något speciellt händer på sidan tex vid klick på ett specifikt ställe eller knapp. Man använder det genom document tagen för att hitta en specifik HTML sektion, lägga till eller ta bort innehåll i HTML tex skapa en ny div eller lägga till en lista inuti en befintlig div, ändra innehållet helt med innerHMTL bla.

Fin kod - Vad menas med det begreppet? Varför är det viktigt?

Med fin kod menas att man ska skriva kod som mer än du själv förstår, den ska vara lättläst och ha en bra struktur. Detta gör man genom att använda namn som beskriver klasser, funktioner, IDn på ett tydligt sätt. Inte använda förkortningar som bara du förstår tex JHL utan då döpa den till jagHeterLukas så andra också kan gå in i min kod och förstår vad den gör. Kommentera koden är också en bra ska att tillämpa för att tydligt kunna förklara olika delar och de delarnas syfte. Även viktigt att ta bort kod som inte används så de inte bara ligger där och blir förvirrande. Varför det är viktigt är för att fler än du själv ska kunna gå in i din kod och kunna jobba med det projektet och veta vad koden står för så koden inte bli oanvändbar för nästa om du tex skulle byta arbetsplats.

Ramverk - Inom webbutveckling använder man begreppet ramverk, vad är det? Beskriv vad det är och beriätta lite mer om två valfria.

Ramverk är något där jag kan plocka färdig kommandon och funktioner utan att behöva göra dom själva. Istället för att behöva uppfina hjulet varje gång så finns det färdig kod som jag kan låna in i mina projekt för att effektivisera mitt arbete. Bootstrap som är ett front-end ramverk som man kan använda sig av för att kunna skapa hemsidor, Bootstrap gör det enkelt att skapa mobilanpassade hemsidor. Sedan finns det även FontAwsome som är ett ramverk som kan användas som för att använda färdiga iconer till din sida tex. iconer för sociala medier, iconer för att kontaktuppgifter osv. Iconer är väldigt talade på en hemsida och kan vid många tillfällen ersätta text.
